# 워드 작성 템플릿 — 법령 원문 + 쉬운 해석 추출용

`npm run parse:guide -- <워드파일>` 로 JSON 초안을 만들 때, 아래 규칙을 지키면 자동 분리가 잘 됩니다.

## 목표 형식 (둘 중 하나)

### A. 줄 단위 — `/guide` 페이지용 (권장, 누락 방지)

조문 제목 줄 다음부터, **원문 조각**과 **쉬운 해석**을 번갈아 넣을 때 아래 **구분 줄**을 사용합니다. 구분 줄은 **해당 줄에 마커만** 오도록 하세요(앞뒤 공백만 허용).

- `<<<원문>>>` — 다음 줄부터 이어지는 텍스트가 **원문** 한 덩어리(여러 줄 가능).
- `<<<쉬운>>>` — 다음 줄부터 이어지는 텍스트가 **쉬운 해석** 한 덩어리(여러 줄 가능).

한 조 안에서 `<<<원문>>>` → `<<<쉬운>>>` → `<<<원문>>>` → `<<<쉬운>>>` … 을 반복합니다. 웹 가이드에서는 각 쌍이 표의 **한 행**이 됩니다.

```text
제1조(목적)

<<<원문>>>
이 법은 주거용 건물의 임대차(賃貸借)에 관하여 「민법」에 대한 특례를 규정함으로써
국민 주거생활의 안정을 보장함을 목적으로 한다.

<<<쉬운>>>
이 법은 ‘살 집 임대차’를 다루고, 민법과 달리 정한 특별 규칙을 둔다는 취지로 이해하면 됩니다.

<<<원문>>>
② … (다음 호·문단 원문)

<<<쉬운>>>
②에 대한 쉬운 설명 …
```

조문 첫 줄 `제N조(제목)` 직후에 본문이 오기 전에 **첫 `<<<원문>>>`** 으로 시작하는 것을 권장합니다. `<<<원문>>>` 앞에 임의 텍스트가 있으면 경고로 잡힙니다.

### B. 조문당 한 덩어리 — 이전 방식(호환)

1. **한 덩어리로** `제N조(한글제목)` 로 시작하는 **실제 법령 원문**(국가법령정보센터 본문과 동일하게 붙여넣기).
2. 그 다음에 **반드시** 마커 줄: `쉬운 해석:` 또는 `쉬운 해석：` (전각 콜론 가능).
3. 마커 뒤에 **쉬운 해석** 본문(여러 문단 가능). 다음 `제…조`가 나오기 전까지가 한 조의 해석입니다.

조문 본문에 `<<<원문>>>` 이 **한 번도 없으면** 파서는 B 방식으로만 처리하고, 결과 `lines` 배열에는 **행이 1개**만 들어갑니다.

### 예시 (B 방식)

```text
제1조(목적) 이 법은 주거용 건물의 임대차(賃貸借)에 관하여 「민법」에 대한 특례를 규정함으로써 국민 주거생활의 안정을 보장함을 목적으로 한다.

쉬운 해석: 이 법은 ‘살 집 임대차’를 다루며, 민법보다 유리하거나 특별한 규칙을 둔다는 취지입니다. 표현은 이해를 돕기 위한 것이며 실제 적용은 원문과 사실관계를 봐야 합니다.
```

## 조문 번호 규칙

- `제1조`, `제3조`, `제6조의2` 처럼 **아라비아 숫자**와 `의` 가 들어간 형태를 지원합니다.
- 괄호 제목은 선택입니다: `제2조(정의)` 또는 `제2조` 로 시작해도 됩니다.

## 쉬운 해석 마커

- 허용: `쉬운 해석:` `쉬운 해석：` (공백은 `쉬운`과 `해석` 사이에만 있어도 됨).
- **새 줄**에서 시작하는 것을 권장합니다. 같은 줄에 원문 끝과 붙어 있으면 분리가 어긋날 수 있습니다.

## 자주 나는 문제

- **민법·다른 법** 인용 안에 `제○조`가 있으면, 워드 본문 전체를 한 번에 넣지 말고 **이 법 조문만** 블록으로 나누는 것이 안전합니다.
- 마커를 빼먹으면 해당 조에 `parseWarnings` 로 “쉬운 해석 마커 없음”이 기록됩니다. `data/source/generated/*-from-guide.generated.json` 을 열어 확인하세요.

## 산출물

- `data/source/raw/<lawId>-guide-raw.json` — 추출된 단락·전체 텍스트(수동 보정용).
- `data/source/generated/<lawId>-from-guide.generated.json` — 조문별 `lines[]`(원문·쉬운 쌍), 합쳐진 `fullText` / `easyInterpretation`, `parseWarnings`.

앱의 **쉬운 해설 페이지**는 `data/guide/housing-rental-guide.json` 을 읽습니다. 파싱 직후 반영하려면:

```bash
npm run parse:guide -- ./dataset/문서.doc --write-guide
```

검수 후 `keywords`, `relatedArticleNos` 등을 보강해 `data/source/housing-rental-protection.json` 에 반영합니다(채팅·조항 목록용).

## 저작권·면책

- 법령 원문: 국가법령정보센터 이용 조건을 따르세요.
- 쉬운 해석: 직접 작성한 설명임을 유지하고, 법률 자문이 아님을 데이터·UI에서 안내합니다.
